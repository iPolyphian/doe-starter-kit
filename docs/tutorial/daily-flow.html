A Day with DOE

What using DOE actually looks like, from first coffee to end of day — hour by hour.

The rhythm of a DOE day
/stand-up → build & chat → /sitrep → build & chat → /wrap
break — context window cleared
/crack-on → build & chat → /wrap → /eod
session start / resume status check save progress day review
ⓘ
Key distinction: /stand-up starts your first session of the day. /crack-on is a resume command — use it when you return after a break or open a new conversation mid-session. They serve different purposes.

Morning — Starting Your Day

Open your project, launch Claude Code, and run /stand-up. This is your daily kick-off — Claude reads STATE.md and todo.md, then presents a briefing card.

Read the briefing carefully — it often surfaces things you'd forget, like a stale blocker or a decision from last week. If the plan looks right, approve it and start building. If you want to change priorities, say so now.

ⓘ
Don't skip the stand-up card. Two minutes reading it now saves twenty minutes of confusion later. It often catches things — a stale blocker, a forgotten decision, a task that's been sitting too long.

Working — Building in Plain English

Once you've approved the stand-up plan, just start talking to Claude. Describe what you want in plain English. No special syntax, no commands — conversation is the interface.

claude — my-project 
> Build the stats card component. It should show total items,
  items this week, and a sparkline. Use the existing CSS variables.

# Claude builds, verifies, commits. You steer — Claude builds.

/wrap updates STATE.md with your current position, records any learnings, calculates session stats, and generates a summary. Without it, the next session's /stand-up won't know what you accomplished. Your code is safe in git either way — but session context is lost without a wrap.

⚠
Most important habit: Always /wrap before ending a session. If you forget, STATE.md won't reflect where you actually are, and the next stand-up works from stale information.

After a Break — Resume with /crack-on

When you return after a break and the context window has been cleared — whether from starting a new conversation, using /clear, or coming back from lunch — use /crack-on to resume.

ⓘ
/crack-on is a resume command. It picks up where you left off, skips the full briefing, and immediately starts building the next step. Use it when you know the plan and just want to get back into it. If you want the full context review first, use /stand-up instead.
claude — my-project 
# New conversation, back from lunch. Context window is clear.
> /crack-on

──────────────────────────────────
  CRACK-ON  14:15  resuming
──────────────────────────────────
  Last wrapped: 13:02  (1h 13m ago)
  Next step: Responsive layout pass

Starting fresh. Full briefing, plan review, explicit approval. Use at the start of a new day or when you want the full picture first.

/crack-on

Resuming after a break. Compact card, no review, immediate start. Use when you've been away and want to get back into it quickly.

Command Reference

The six commands that cover the full daily cycle.

/stand-up Start a new session

Reads STATE.md and todo.md, presents a briefing card, and waits for your approval before starting. Use at the beginning of the day or any time you want the full context briefing first.

/sitrep Quick status snapshot

Shows completed steps, current step, and what's next. Read-only — safe to run any time, as many times as you like. No changes to state.

/wrap Save session progress

Updates STATE.md, records learnings, calculates session stats, and generates a summary. Run before every break. The most important habit in DOE.

/crack-on Resume after a break

Reads state files, shows a compact card, and immediately continues building. Use when you've been away and want to resume — no review, no approval needed. For a full briefing, use /stand-up instead.

/eod End-of-day summary

Shows everything you shipped today, across all sessions. Timeline of sessions, aggregate metrics, and current project status. Run after your final /wrap.

/hq Portfolio view

Project dashboard: all-time session count, streak, features shipped, this week's activity, and roadmap status. Run weekly or whenever you want the big picture.

Key Habits

The habits that make the biggest difference, distilled from the daily routine above.

Always /wrap before ending a session

Without it, STATE.md won't reflect where you actually are, and the next stand-up works from stale information. Your code is safe in git either way — but session context is lost without a wrap.

One task per session

Context pollution is real. When a conversation accumulates unrelated topics, Claude's responses get less focused. Keep each session about one thing. If you need to switch topics, /wrap then /clear first.

Read the stand-up card

Don't skip past it. It often surfaces things you'd miss — a stale blocker, a forgotten decision, a task that's been sitting in the queue too long. Two minutes of reading saves twenty minutes of confusion.

Failures make the system smarter

DOE has a self-annealing system: when something fails, Claude diagnoses the root cause, fixes it, and logs the learning so it doesn't happen again. You don't need to manage this — just know it's happening, making every future session better.

Trust the rhythm

Stand-up, build, wrap. It feels like overhead the first few times, but it pays for itself quickly. The system gets smarter about your project with every session — and that only works if it has clean state to work from.

ⓘ
These are tools, not rules. You don't have to follow this schedule exactly. But /stand-up at the start and /wrap at the end are the two that matter most for keeping the project's memory intact.
