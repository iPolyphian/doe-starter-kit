Frequently Asked Questions

Quick answers with links to detailed explanations

Setup Problems

Claude won't start — "command not found"

Claude Code needs to be installed globally. Make sure you ran npm install -g @anthropic-ai/claude-code and your terminal can find it.

Read more → Installation guide

A hook is blocking my commit

Hooks are safety checks. Read the error message — it usually tells you exactly what to fix. Common causes: missing STATE.md update, no commit message.

Read more → Tips & Mistakes — Understanding Hooks

Git errors during setup

Make sure git is installed (git --version). If you see "not a git repository", run git init in your project folder first.

Read more → Installation guide

setup.sh failed or won't run

On macOS, you may need to run chmod +x setup.sh first. If it fails partway through, it's safe to run again — it won't duplicate anything.

Read more → Installation guide

Session Problems

Context is running out mid-task

Run /wrap to save progress, then start a fresh session with /stand-up or /crack-on. One task per session prevents this.

Read more → Context & Sessions

Claude seems confused or is going in circles

Run /clear to reset the conversation. If the problem persists, check STATE.md for stale information.

Read more → Tips & Mistakes — When Things Go Wrong

I lost work — Claude's changes disappeared

Check git log — every completed task creates a commit (save point). Use git diff to see what changed.

Read more → Tips & Mistakes — When Things Go Wrong

Claude made unwanted changes to my code

Run git diff to see what changed, then git checkout -- filename to undo specific files. For bigger rollbacks, see break-glass.md.

Read more → Tips & Mistakes — When Things Go Wrong

Framework Problems

Slash commands aren't working (/stand-up, /wrap, etc.)

Commands live in .claude/commands/. Make sure setup.sh copied them. Run ls .claude/commands/ to check.

Read more → Commands reference

Files are ending up in the wrong places

DOE has a strict directory structure. Check CLAUDE.md for the correct locations.

Read more → DOE Architecture

How do I update to a newer DOE version?

Run /pull-doe to pull the latest starter kit improvements into your project. You don't need a GitHub account — the command handles everything.

Read more → Tips & Mistakes — sync/pull section

What does this error message mean?

Most errors include the file and line number. Read the full message before asking Claude to fix it. Claude is good at explaining errors if you paste them.

Read more → Tips & Common Mistakes
