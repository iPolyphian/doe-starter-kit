IDE Compatibility

DOE works with any editor. Here's how to avoid conflicts with AI-powered IDEs.

DOE Is IDE-Agnostic

DOE runs entirely through Claude Code CLI, git, the file system, and Python scripts. Your IDE is just a text editor with a terminal. Everything that makes DOE work — CLAUDE.md, hooks, contracts, slash commands, directives, execution scripts, stats tracking, session lifecycle — is independent of which editor you use.

VS Code, Cursor, Windsurf, Zed, Neovim, plain terminal — all work identically. DOE doesn't read your IDE settings, doesn't install editor plugins, doesn't depend on any editor API. If you can type in a file and run a terminal command, you have everything DOE needs.

ⓘ
The test: if you can open a terminal and run claude, DOE works in your editor.

The Problem with AI-Powered Editors

DOE's architecture has a core principle: "Probabilistic AI handles reasoning. Deterministic code handles execution." Claude Code is the probabilistic layer, tightly integrated with the deterministic layer — hooks, contracts, scripts, commands. Together they form a closed loop where every AI action is tracked, verified, and auditable.

AI-powered editors like Cursor, Windsurf, and GitHub Copilot Workspace introduce a second probabilistic layer with zero integration to DOE's deterministic enforcement. When Cursor's Composer or Windsurf's Cascade edits a file:

  • No PostToolUse hooks fire
  • No contract verification
  • No step tracking or session continuity
  • No CLAUDE.md directives loaded
  • No learnings.md consulted
  • No commit discipline enforced

It's a capable AI operating entirely outside your guardrail system.

Recommended Setup

ⓘ
  1. Use the AI editor as a better text editor — Cursor and Windsurf are genuinely nice editors with good UX. That value is real.
  2. Keep Tab autocomplete on — It's the one AI feature that's purely additive. Doesn't interfere with DOE.
  3. Disable multi-file agents (Composer, Cascade) — Direct conflict with Claude Code. In Cursor: Settings > Features > Composer > disable.
  4. Disable inline AI edit (Cmd+K) — Or restrict to scratch files only. In Cursor: Settings > Features > disable Cmd+K.
  5. AI Chat: read-only rule — Use for quick lookups ("what does this function do?"). Never let it write code.
  6. Don't create project rules files — No .cursorrules, no .windsurfrules. CLAUDE.md is the single source of truth.
  7. All code changes go through Claude Code — This preserves the full DOE pipeline: hooks, contracts, session tracking, learnings.

Cursor Quick Start

Cursor is a VS Code fork with built-in AI. It's a genuinely nicer editor — faster, better UI, good autocomplete. Here's how to set it up for DOE without any conflicts.

1. Open your project

File > Open Folder > select your project. Everything appears exactly as it did in VS Code — same file tree, same extensions, same git integration. Nothing to migrate.

2. Set the layout to Editor mode

Click the gear icon (top-right) and select Editor, not Agent. Editor mode is the traditional code editor layout — file tree on the left, editor in the centre. Agent mode puts Cursor's AI chat front and centre, which you don't want.

3. Move the terminal to the right

Press Cmd+J to open the terminal, then right-click the terminal title bar and select Move Panel Right. This docks the terminal alongside your code instead of below it. Alternatively: Cmd+Shift+P > "View: Set Panel Position Right".

4. Start Claude Code

In the terminal, type claude and press Enter. That's it. Full Claude Code with all your DOE setup — hooks, commands, contracts, directives, everything. The terminal in Cursor is identical to VS Code's terminal. Claude Code doesn't know or care which editor hosts it.

5. Leave Cursor Tab on

You'll see "Cursor Tab" in the bottom-right status bar — this is AI autocomplete. It's the one Cursor AI feature that's safe and useful with DOE. It suggests code as you type, doesn't modify files autonomously, and doesn't interfere with Claude Code. The free tier gives 2,000 completions per month, which is plenty when Claude Code writes most of your code.

6. Ignore the chat panel

The right panel labelled "New Chat" with "Agent" / "Auto" / "Local" is Cursor's AI agent. You don't need it — Claude Code in the terminal is your AI. If the chat panel is open, just close it. If you accidentally open it with Cmd+L, close it again. The free tier only gives 50 slow AI requests per month anyway, so it's self-limiting.

ⓘ
What "Agent", "Auto", and "Local" mean in Cursor's chat panel:
  • Agent — the chat mode. Agent is autonomous (reads/writes files, runs commands). Other options: "Ask" (read-only Q&A) and "Manual" (targeted edits). None of these integrate with DOE.
  • Auto — the model selector. "Auto" means Cursor picks the model. You can also manually select Claude Sonnet, GPT-4o, etc. Irrelevant if you're not using the chat.
  • Local — means the agent runs on your machine (foreground). Cursor Pro has "Background Agents" that run in the cloud. On the free tier, only Local is available.

7. Free tier is all you need

Your daily workflow

Identical to VS Code. The only difference is occasional Tab completions when you manually type code.

Cmd+J          → open terminal
claude         → start Claude Code
/stand-up      → kick off session
/crack-on      → start building
...            → all DOE workflow as normal
/wrap          → end session

The Context Split Problem

DOE has session continuity. /stand-up loads STATE.md and todo.md. /crack-on reads the plan and starts a step. Each interaction builds on prior context. /wrap closes the session and updates everything. The AI-powered editor's chat and agent has none of this context. Every interaction is a cold start.

If you split attention between Claude Code and the editor's AI, you fragment your own mental model too. "Did I tell Claude Code about that change or was it a Cmd+K edit?" becomes a real question. The answer matters — Claude Code tracks what it changed, but it has no visibility into edits made by a different AI agent. Those edits exist in your files but not in DOE's understanding of your project state.

Keeping one AI agent as the single driver avoids this entirely. Use the editor for what editors are good at — navigating files, reading code, typing — and let Claude Code handle the reasoning and execution pipeline.

Cost Considerations

Using an AI editor alongside Claude Code means paying for two AI systems:

  • Your Claude plan — powers Claude Code, unchanged regardless of editor
  • The editor subscription — Cursor Pro $20/mo, Windsurf Pro $15/mo, etc.

If you disable the editor's agent features and only use Tab autocomplete, the free tier of most AI editors is sufficient. No need to pay for features you've disabled. Cursor Free gives 2000 Tab completions per month. That's plenty when Claude Code does most of the writing.
